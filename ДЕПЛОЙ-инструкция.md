# Deutsch Meister — как деплоить (памятка)

Коротко: у проекта **две независимые части**, деплоятся по-разному.

| Что | Где живёт | Как деплоится |
|-----|-----------|----------------|
| **Сайт / мини-апп** (HTML, CSS, `js/*`, уроки) | GitHub Pages → https://vanguar.github.io/deutsch-meister/ | **Автоматически** при `git push` в ветку `main` |
| **Бэкенд** (папка `api/` — бот, озвучка, прогресс) | Vercel → https://deutsch-meister-puce.vercel.app | **Вручную** командой `vercel --prod` (от `git push` НЕ деплоится!) |

---

## 1. Деплой сайта (фронтенд)

Просто пушишь в GitHub — Pages сам пересоберёт за 1–2 минуты.

```bash
git add .
git commit -m "что изменил"
git push origin main
```

## 2. Деплой бэкенда (папка `api/`)

Если правил файлы в папке **`api/`** (например `api/webhook.py`), нужно **отдельно** задеплоить на Vercel:

```bash
vercel --prod
```

> ⚠️ Если `vercel` не установлен: `npm i -g vercel`, затем `vercel login`.

Проверка, что бэкенд обновился:
```bash
curl https://deutsch-meister-puce.vercel.app/api/webhook
# должно вернуть JSON с "service": "webhook"
```

---

## 3. ВАЖНО: сброс кэша при изменении JS/CSS

У сайта есть Service Worker, который кэширует файлы. Чтобы правки JS/CSS **точно доехали** до пользователей, при изменении любого скрипта надо:

1. Поднять версию `?v=N` у этого файла **во ВСЕХ 32 файлах уроков** (`lessons/**/index.html`) и в `index.html`.
   Пример: `js/tts.js?v=16` → `js/tts.js?v=17`.
2. Поднять номер кэша в `service-worker.js`: `const CACHE = 'deutsch-meister-vNN'` (увеличить NN).

Без этого пользователи будут видеть старую версию из кэша.

**Как обновить на телефоне:** полностью закрыть мини-апп в Telegram и открыть заново (Service Worker обновится и страница один раз перезагрузится). Если упрямо — Telegram → Настройки → Данные и память → Очистить кэш.

## 4. Как устроен звук (озвучка) — на всякий случай

- **В браузере / PWA:** нативный голос телефона (`speechSynthesis`).
- **В Telegram:** нативный голос молчит, поэтому звук идёт через **Web Audio API**: фронтенд скачивает mp3 с нашего прокси `api/` → `/api/tts` (он берёт озвучку у Google Translate TTS и добавляет CORS-заголовок) → декодирует и играет.
- Прокси нужен потому, что Google TTS не отдаёт CORS, а для Web Audio он обязателен.
- Прежний источник **StreamElements сломался** (отдаёт 401) — не возвращаться к нему.

Если однажды звук снова пропадёт в Telegram — первым делом проверь, что `/api/tts` отдаёт mp3:
```bash
curl -s -o test.mp3 -w "%{http_code} %{content_type} %{size_download}\n" \
  "https://deutsch-meister-puce.vercel.app/api/tts?tl=de&text=Hallo"
# ждём: 200 audio/mpeg <размер>0
```

## 5. Эндпоинты бэкенда (всё в одном файле `api/webhook.py`)

Все `/api/*` идут в один обработчик, который сам разбирает путь:

- `/api/webhook` — Telegram-бот.
- `/api/tts?text=...&tl=de` — прокси озвучки (mp3 + CORS).
- `/api/progress` — облачный прогресс (работает только если в Vercel заданы переменные `UPSTASH_REDIS_REST_URL` и `UPSTASH_REDIS_REST_TOKEN`; сейчас прогресс локальный, в браузере).

---

## 6. Опционально: автодеплой бэкенда от пуша

Сейчас Vercel не связан с GitHub, поэтому деплой ручной. Чтобы включить автодеплой:
Vercel dashboard → проект **deutsch-meister** → **Settings → Git → Connect** → GitHub → репозиторий `vanguar/deutsch-meister`, production branch = `main`.
После этого `git push` будет деплоить и бэкенд.
